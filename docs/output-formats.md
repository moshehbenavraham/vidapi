# Output Formats

VidAPI renders one requested output artifact per render. Editly remains the
default renderer and produces an MP4 intermediate. Non-MP4 formats are finished
from that intermediate with FFmpeg before the artifact is published.

## Presets

Named presets provide stable defaults for common publishing targets:

| Preset | Width | Height | FPS | Quality |
|--------|-------|--------|-----|---------|
| `tiktok` | 1080 | 1920 | 30 | `high` |
| `reels` | 1080 | 1920 | 30 | `high` |
| `shorts` | 1080 | 1920 | 30 | `high` |
| `youtube` | 1920 | 1080 | 30 | `high` |
| `square-ad` | 1080 | 1080 | 30 | `medium` |
| `preview-low` | 640 | 360 | 24 | `low` |

Explicit `output.width` and `output.height` always take precedence over preset
dimensions. Presets only fill omitted fields.

## Formats

Supported output formats are `mp4`, `webm`, `gif`, and `png-sequence`.

| Format | Download Suffix | Media Type | Artifact Shape |
|--------|-----------------|------------|----------------|
| `mp4` | `.mp4` | `video/mp4` | Single file |
| `webm` | `.webm` | `video/webm` | Single file |
| `gif` | `.gif` | `image/gif` | Single file |
| `png-sequence` | `.zip` | `application/zip` | Zip archive plus manifest |

PNG sequence outputs use deterministic frame names and a JSON manifest. The
download endpoint returns the archive so the existing render download contract
continues to work.

## Metadata

Succeeded render responses and webhook payloads include output metadata:

- `format`
- `media_type`
- `filename`
- `frame_count`
- `manifest_url`

Durable metadata stores stable format facts and artifact keys only. Public,
signed, and proxied URLs are generated by `StorageUrlResolver` at response or
webhook time.

## Guardrails

Format-specific limits protect the worker from unbounded post-processing:

- GIF and PNG sequence output must fit configured duration, FPS, resolution, and
  frame count limits.
- FFmpeg post-processing uses an explicit timeout and bounded stderr capture.
- API admission and worker pre-flight both revalidate renderer and format
  constraints before work begins.
